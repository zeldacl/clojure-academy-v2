# 注册系统维护手册

## 系统职责

注册系统负责方块、物品、能力、网络消息等元数据到平台注册表的映射，确保 ID 稳定与生命周期正确。

## 模块边界

- `mcmod/registry`：注册抽象与元数据组织。
- `ac/registry`：业务条目装配与声明。
- `forge1201/registry.clj`：平台注册时机与落地执行。

## 运行时流程

1. `ac` 组织内容元数据。
2. 平台层按生命周期将元数据落地到 Forge 注册流程。
3. 后续系统通过稳定 ID 访问注册结果。

## 扩展点

- 新可注册类型优先扩展元数据模型，再补平台适配。
- 新注册流程应与现有生命周期钩子对齐。

## 排障手册

- 条目未注册：检查元数据声明、命名空间与注册事件顺序。
- 重名冲突：检查 `mod_id:path` 唯一性与重复装配。
- 运行时找不到条目：检查是否跨侧误访问或初始化过早。

## 变更风险

- 改动 registry 名称会破坏存档与跨模组兼容。
- 注册时机错位会触发启动阶段错误。

## 兼容性约束

- 将 registry ID 视为稳定 API，避免随意重命名。
- 变更命名规则时需提供迁移策略与风险说明。
