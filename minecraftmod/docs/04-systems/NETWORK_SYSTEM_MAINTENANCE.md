# 网络系统维护手册

## 系统职责

网络系统负责客户端与服务端间的消息通道、消息编解码与处理分发，保障状态同步的正确性与最小开销。

## 模块边界

- `mcmod/network`：网络抽象与通用消息协议。
- `forge1201/events` 等：平台网络注册与消息接入。
- `ac` 只定义业务消息语义，不承担底层通道实现。

## 运行时流程

1. 启动时注册消息类型与处理器。
2. 事件或业务动作触发消息发送。
3. 对端解码并在正确线程执行处理逻辑。

## 扩展点

- 新消息类型通过统一注册入口添加。
- 大对象同步优先拆分为增量消息与快照消息。

## 排障手册

- 消息未到达：检查注册表、消息 ID、通道初始化时机。
- 消息处理报错：检查编解码字段一致性与线程上下文。
- 带宽/性能问题：检查高频广播与冗余同步字段。

## 变更风险

- 消息字段改动可能破坏联机兼容。
- 错误线程执行会导致状态竞争或崩溃。

## 兼容性约束

- 消息协议变更需保持版本兼容或明确中断版本线。
- 网络处理必须遵守服务端/客户端线程约束。
