# UI 系统维护手册

## 系统职责

UI 系统负责界面声明、容器交互、客户端渲染与服务端状态同步。目标是确保交互逻辑与渲染逻辑分层、可维护、可排障。

## 模块边界

- `mcmod/gui`：通用 GUI 抽象、DSL 与容器协议。
- `ac/gui`：业务界面装配与交互逻辑。
- `forge1201/gui`：平台容器与客户端绑定实现。
- 客户端渲染代码仅放在 client 侧目录。

## 运行时流程

1. 服务端打开容器并创建会话上下文。
2. 网络层同步必要状态到客户端。
3. 客户端基于 DSL/组件渲染界面并处理输入。
4. 输入事件回传服务端并更新状态。

## 扩展点

- 新界面优先复用通用容器与组件协议。
- 新交互能力优先在 `mcmod/gui` 抽象，再由 `ac/gui` 组合。
- 新同步字段通过统一消息层扩展，不做散乱直连。

## 排障手册

- UI 打不开：检查菜单注册、打开入口与侧分离。
- UI 打开但无数据：检查服务端状态填充和首次同步消息。
- UI 卡顿：检查每帧重算与高频网络同步。

## 变更风险

- 容器协议字段变更会影响客户端兼容。
- 渲染层误引入服务端逻辑会导致 Dedicated Server 崩溃风险。

## 兼容性约束

- 保持容器消息格式向后兼容。
- 对外 DSL 字段与语义保持稳定。
