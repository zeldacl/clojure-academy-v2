# DSL 基础设施维护手册

## 系统职责

DSL 基础设施负责把内容定义转换为统一元数据与运行时行为，覆盖 Block/Item/NBT/Tile 等声明式开发能力。

## 模块边界

- `mcmod/block`、`mcmod/item`、`mcmod/nbt`：DSL 核心定义与转换逻辑。
- `ac/block`、`ac/item`：业务内容声明与组合。
- 平台层负责消费元数据，不复制 DSL 规则。

## 运行时流程

1. 业务层通过 DSL 定义内容。
2. DSL 产出标准元数据/处理器。
3. 注册系统与平台层消费这些元数据完成落地。

## 扩展点

- 新 DSL 关键字需先定义语义与约束，再实现解析。
- 新 DSL 功能优先复用已有元数据模型，减少格式分裂。

## 排障手册

- DSL 无效：检查宏展开结果与元数据结构。
- 注册后行为异常：检查 DSL 到注册层的字段映射。
- 维护困难：检查是否绕过 DSL 直接写平台特化逻辑。

## 变更风险

- DSL 语义调整会影响大量既有内容定义。
- 元数据字段改动会影响注册与运行时消费方。

## 兼容性约束

- 保持 DSL 关键字和核心语义稳定。
- 语义变更需提供兼容层或迁移说明。
