# 无线系统维护手册

## 系统职责

无线系统负责节点网络、信号/能量传播与终端交互入口，强调“拓扑建模、状态一致性、性能可控”。

## 模块边界

- `ac/wireless`：核心网络模型与业务规则。
- `ac/terminal`：终端交互与运维入口。
- 平台适配只负责生命周期与事件接入，不承载无线业务策略。

## 运行时流程

1. 节点注册到网络模型。
2. 网络建立连接关系并计算传播结果。
3. 终端/界面层读取网络快照并执行运维操作。
4. 变更通过事件驱动回写网络状态。

## 扩展点

- 新网络规则以策略函数扩展，避免分支散落。
- 新节点类型通过统一注册与能力声明接入。
- 运维命令与 UI 操作复用同一服务层入口。

## 排障手册

- 节点不入网：检查节点注册、维度/坐标上下文和生命周期回调。
- 网络状态错乱：检查增删节点时的拓扑重建与缓存刷新。
- 性能下降：排查高频轮询，优先改为事件触发与增量更新。

## 变更风险

- 拓扑算法改动可能影响大规模网络稳定性。
- 节点状态结构改动可能影响旧存档读取。

## 兼容性约束

- 节点 ID 和网络标识保持稳定。
- 网络持久化字段变更需提供兼容读取逻辑。
