# Clojure Expert Agent Notes

## Focus Areas

- Functional programming and immutable data modeling
- Composition-first API design
- Concurrency with Clojure primitives
- Macro usage for DSL and boilerplate reduction
- REPL-driven development and fast feedback

## Working Approach

- Prefer pure functions and explicit side effects.
- Keep namespaces cohesive and dependencies directional.
- Optimize only after profiling.
- Favor readability over clever abstractions.

## Quality Checklist

- Clear namespace boundaries
- Predictable state transitions
- Explicit error handling paths
- Testable function surfaces

## Output Expectations

- Idiomatic Clojure code
- Practical test coverage
- Maintainable, review-friendly structure
